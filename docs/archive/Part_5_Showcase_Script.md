# Assignment 7: Part 5 Submission (SentryC2 Visualization Update)

For this phase of the project, the SentryC2 Manufacturing Execution System (MES) was entirely refactored to support a complex 27-node triangular branch-and-merge topology tracking real-time layout movement. The front-end interface was rebuilt using a natively populated SVG graph overlaid on responsive CSS geometry. This allows the system to accurately map active transitions of Work-In-Process (WIP) across physical space. The updated graph architecture visibly delineates specialized machine blocks (e.g., CNC Mills and Lathes) from automated robotic handling nodes, visually animating telemetry in real-time as jobs flow seamlessly toward QA Inspection points and Output silos.

To meet the deterministic routing and queueing theory bounds of the assignment, the Python back-end was restructured to move away from naive First-In-First-Out (FIFO) routing into a **Shortest Processing Time (SPT)** dispatching algorithm, which minimizes overall flow time and reduces active WIP volume on the network. Furthermore, rigorous capacity constraints were injected into the system alongside high-variance operational modeling. The CNC Lathe processing cycles now utilize a custom lognormal distribution possessing a high Coefficient of Variation (CV > 1.0) to successfully simulate an M/G/1 queueing environment against random incoming job routes.
